# Установка BPM на сервер (простыми словами)

Эта инструкция для того, кто разворачивает систему у клиента и **не занимается
DevOps**. Никакого CI/CD — только понятные шаги. Рекомендуемый способ — Docker:
он сам ставит нужные версии Python, Node и PostgreSQL, вам не надо настраивать
их вручную.

---

## Что вообще разворачивается

Система состоит из трёх частей — в Docker это три «контейнера»:

1. **База данных** (PostgreSQL) — где хранятся документы и справочники.
2. **Бэкенд** (сервер логики, FastAPI) — обрабатывает запросы.
3. **Фронтенд** (веб-интерфейс) — то, что видит пользователь в браузере.

Пользователь открывает сайт → фронтенд → бэкенд → база. Всё это поднимается
**одной командой**.

## Способ 1 (рекомендуется): Docker

### Шаг 1. Сервер

Возьмите виртуальный сервер (VPS) у любого хостинга. Для начала хватит:
2 ядра, 4 ГБ памяти, 40 ГБ диска, ОС **Ubuntu 22.04/24.04**. Получите его
IP-адрес и доступ по SSH (логин/пароль или ключ).

### Шаг 2. Установить Docker (один раз на сервере)

Подключитесь к серверу по SSH и выполните:

```bash
curl -fsSL https://get.docker.com | sh
```

Проверка: `docker --version` должен показать версию.

### Шаг 3. Скачать код

```bash
git clone https://github.com/DamirKZ82/bpm.git
cd bpm
```

(Или загрузите архив с кодом и распакуйте — как удобнее.)

### Шаг 4. Заполнить настройки

Скопируйте шаблон и откройте его в редакторе:

```bash
cp backend/.env.example backend/.env
nano backend/.env
```

Обязательно задайте:
- `DB_PASS` — придумайте сложный пароль базы (тот же укажите при желании в
  `docker-compose.yml`, либо оставьте по умолчанию и просто пропишите здесь);
- `JWT_SECRET` — длинная случайная строка. Сгенерировать можно так:
  `openssl rand -base64 48`;
- `APP_BASE_URL` и `CORS_ORIGINS` — адрес, по которому будет открываться система
  (например `https://bpm.клиент.kz`, а до домена — `http://IP-адрес`).

`DB_HOST` оставьте `db` — это имя контейнера базы.

Почту и Telegram можно оставить пустыми и настроить позже.

### Шаг 5. Запустить

```bash
docker compose up -d --build
```

Первый запуск займёт несколько минут (скачивание и сборка). Дальше система
поднимется сама, миграции базы применятся автоматически.

Откройте в браузере `http://IP-адрес-сервера` — увидите вход в BPM.

### Шаг 6. Создать первого администратора

Пока не подключён вход через AD, заведите dev-администратора:

```bash
docker compose exec backend python scripts/seed_dev.py
```

После этого можно войти и вести справочники/пользователей из интерфейса.

## Как обновить систему до новой версии

```bash
cd bpm
git pull
docker compose up -d --build
```

Данные в базе и файлы сохраняются (они в «томах» Docker, а не в контейнерах).

---

## HTTPS (замок в браузере) — по домену

Когда у клиента есть домен (например `bpm.клиент.kz`), направьте его A-запись
на IP сервера и включите бесплатный сертификат. Самый простой путь — поставить
перед системой **Caddy** (он сам получает сертификат Let’s Encrypt), либо nginx
+ certbot. Если нужно — подготовлю отдельную короткую инструкцию под ваш домен.

До настройки домена система работает по `http://IP` — этого достаточно для
проверки и демонстрации.

## Резервные копии (важно!)

Раз в сутки делайте копию базы и файлов. Простейший вариант — по расписанию
(cron) выполнять:

```bash
# дамп базы
docker compose exec -T db pg_dump -U bpm bpm > /backup/bpm-$(date +%F).sql
```

Файлы-вложения и логотип лежат в томе `backend_storage` (или в S3, если
настроите облачное хранилище в «Настройки BPM»). Копию тома можно снять так:

```bash
docker run --rm -v bpm_backend_storage:/data -v /backup:/out alpine \
  tar czf /out/storage-$(date +%F).tgz -C /data .
```

Храните копии на отдельном диске/облаке.

## Способ 2 (без Docker): вручную

Если Docker недоступен, всё то же ставится руками (сложнее, но возможно):

1. Установить **PostgreSQL 15**, создать базу `bpm` и пользователя.
2. Установить **Python 3.14**, в папке `backend/`:
   `pip install -r requirements.txt`, заполнить `.env` (`DB_HOST=localhost`),
   выполнить `alembic upgrade head`, запустить
   `uvicorn app.main:app --host 0.0.0.0 --port 8000` как системную службу
   (systemd на Linux).
3. Установить **Node 20**, в папке `frontend/`: `npm ci && npm run build` —
   получится папка `dist/`.
4. Установить **nginx**: отдавать `frontend/dist` как статику и проксировать
   `/api` на `http://localhost:8000` (пример конфига — в `frontend/nginx.conf`).

Docker избавляет от шагов 1–4, поэтому для продажи разным клиентам он удобнее.

## Частые вопросы

**Нужен ли CI/CD?** Нет. CI/CD — это автоматический деплой при изменении кода,
полезен при частых обновлениях своими силами. Для установки клиенту достаточно
ручного `docker compose up -d --build`. Автоматизацию можно добавить позже.

**Где менять логотип и название?** В самой системе: «Администрирование →
Настройки BPM → Брендирование». В код лезть не нужно.

**Что отдавать клиенту?** Отдельную копию кода (не доступ к вашему личному
GitHub) + этот файл. Пароли (`.env`) у каждого клиента свои.

**Порт 80 занят?** Поменяйте в `docker-compose.yml` строку `"80:80"` на,
например, `"8080:80"` — система откроется по `http://IP:8080`.
